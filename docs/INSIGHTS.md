# Insights Log

Product learnings, one dated entry each. Graduation-gate metric 5 counts these (target ≥50).

- **2026-06-09 · Plan-grilling session (pre-code).** Walked all six planning docs against each other before Phase 0; 12 contradictions/gaps resolved and propagated. The ones that were *product* learnings, not just spec fixes:
- **2026-06-09 · Queue noise shapes every ingest decision.** First-connect backfill ingests 30 days of episodes but deliberately extracts nothing from them — R3 (queue noise kills the habit) outweighs day-1 ledger seeding. In-flight commitments enter manually.
- **2026-06-09 · Snooze is a predicate, not a state.** Any status that needs a system waker violates "state advances only by user disposition." Encoding snooze as `snoozed_until` + WHERE clauses removed a job, a status value, and an invariant exception at once. Pattern worth reusing: prefer time-predicates over states that require schedulers.
- **2026-06-09 · Artifacts must pre-sync.** Every brief fires outside the ingest window (7 AM, Sunday 7 PM); without an inline pre-sync step the Monday brief renders Friday's calendar. "Proactive" means fresh-at-generation, not fresh-at-last-poll.
- **2026-06-09 · R2 decided: consumer Gmail + Testing-status OAuth app.** Multiple Gmail accounts are an expected growth path, so the per-mailbox Workspace escape doesn't scale. Weekly token expiry is now a designed-for ritual (one-tap re-consent + push alert in MC-101), not a risk.
- **2026-06-09 · Chat is capture first, retrieval later.** v1 chat = capture surface with inline candidates (Phase 1); ask-the-ledger waits until the substrate it queries exists (Phase 4). Matches the thesis: chat is bolted on.
- **2026-06-09 · Eval truth lives in git, not the DB.** Committed results files per prompt version; `prompt_versions` rows written only at activation. Keeps prod credentials out of CI and dev eval spend out of the R5 cost ticker.
- **2026-06-09 · BullMQ forbids `:` in custom jobIds.** The backlog's `morning-brief:<date>` convention can't be the literal BullMQ id (`:` is its key separator); deterministic ids use `-` (`morning-brief-2026-06-09`) while DB dedupe keys keep the `kind:<key>` shape. Doc convention and runtime constraint now diverge deliberately — `cadence_runs.job_id` stores the real BullMQ id.
- **2026-06-09 · Repeatable schedulers can't carry deterministic per-run ids.** The cron tick is a separate tiny job that enqueues the real job with its natural-key id (tick → `morning-brief-<date>`). This two-step shape is what makes "kill the worker mid-run, restart, no duplicate brief" hold — verified in the Phase-0 drill.
- **2026-06-09 · Per-channel delivery outcomes live in `run_steps`, not run meta.** Email mirror failure fails the run (it's the contract); push failure only writes a failed `push` step — degraded delivery is visible without violating "meta is never rewritten." The append-only design answered the question before it became a schema change.
- **2026-06-09 · Local-machine reality: native Postgres owned IPv4 :5432.** Docker only bound the IPv6 side, so `localhost` silently hit the wrong Postgres with a confusing auth error. Compose now maps 5433 and every default connection string follows. Worth remembering for any future "password authentication failed" on a fresh container.
- **2026-06-09 · iOS install detection falls out of feature detection.** Safari-outside-PWA has no `PushManager`, so "needs install" is just the unsupported branch on an iOS UA — no user-agent sniffing gymnastics needed for the Add-to-Home-Screen prompt.
- **2026-06-09 · Reauth alerts must fire on the flip, not the state.** A `reauth_required` account would otherwise push-notify every 15-minute tick (96 alerts/day). The handler distinguishes "was active and invalid_grant surfaced during THIS run" (alert once, fail run) from "already flagged at job start" (fail fast via `UnrecoverableError`, no Google call, no re-alert). Pattern: notify on transitions, render states.
- **2026-06-09 · The episode guard is the real idempotency layer for prompt evolution.** `extraction_hash` dedupes identical wording, but a v2 prompt wording the same obligation differently would re-flood the queue — the guard (skip episodes that already have commitments) is what makes prompt bumps safe. Tested with a deliberately reworded mock: zero model calls on the second pass.
- **2026-06-09 · Skips are not failures — and don't deserve rows.** The ingest tick outside working hours writes no `cadence_runs` row at all (vs. 96 noise rows/day drowning `/runs`). Invariant 7 is about failures being loud, not about every non-event being logged.
- **2026-06-09 · GCal episodes don't feed extraction in Phase 1.** MC-103 deliberately omits it: calendar chatter is the canonical hard-negative class (R3), and the meeting itself is already structured data in `calendar_events`. If reconciliation later starves for calendar evidence, revisit — the seam is one `extractEpisodeIds` list.
- **2026-06-09 · v1 eval fixtures are synthetic-but-realistic.** The implementing agent has no access to real mail, so the 26-fixture baseline (35% hard negatives) encodes the spec's failure taxonomy instead: weekday math, owner-sent direction flips, buried commitments, scheduling chatter, promo "we'll migrate your data" traps. Fixture realism grows from production rejections per EVAL-SPEC §1.3 — divergence between harness precision and production confirm-rate is the drift signal.
- **2026-06-09 · Owner-sent mail flips person resolution.** Resolving the *sender* on every message would create a person row for Mark himself on every sent email; the counterparty for owner-sent mail is the first recipient. Direction extraction needed the same owner-identity awareness in the prompt (OWNER header with all known addresses).
- **2026-06-09 · PowerShell 5.1 `Set-Content -Encoding utf8` writes a BOM that breaks tsx's package.json parser.** npm tolerates it; tsx does not. File edits on Windows go through the agent's Write tool (BOM-less) — never shell redirection.
- **2026-06-09 · Packet determinism dies by a thousand `now()`s.** The first ContextPacket draft used SQL `now()` in the snooze-wake predicate — same DB state, two assemblies, different packets. Every time-dependent comparison in assembly now takes the job's `now` parameter; the determinism test seeds both sides of the snooze boundary. Rule: a packet input is either a row or an argument, never the wall clock.
- **2026-06-09 · JSONB round-trips reorder keys — byte-determinism is an assembly property, not a storage property.** `context_packets.content` comes back with keys re-sorted, so "same inputs → byte-identical packet" is asserted on the returned object; the stored row gets deep-equality. Anyone diffing packets must diff parsed JSON, not strings.
- **2026-06-09 · Adapter factories are an invariant leak if the barrel exports them.** Exporting `createVoyageAdapter` from `packages/llm` would let callers embed without a `model_calls` row — the lint rule only catches SDK imports, not factory bypasses. Same treatment as the Anthropic adapter now: internal module, never in the barrel.
- **2026-06-09 · pgvector HNSW post-filters WHERE clauses.** The cosine index can't pre-filter on `owner_id/status/pinned`, so filtered top-k may degrade at corpus scale; harmless at single-user size but the comment in `retrieveMemories` now says so explicitly to stop a Phase-4 false assumption.
- **2026-06-09 · Per-channel retry = re-read the artifact's lifecycle columns at attempt start.** The notify retry needed no state machine: each attempt re-reads `emailed_at`/`pushed_at` and skips delivered channels (a `skipped` run step, visible). The append-once columns double as the retry ledger — idempotency fell out of the existing schema.
- **2026-06-09 · "Why did you say this?" must survive schema generations.** The debug view crashed on Phase-0 hello packets (no `commitments` array) — old artifacts outlive the code that wrote them. Traceability views guard every field they walk; immutable artifacts mean readers, not writers, absorb schema drift.
- **2026-06-09 · The presync step is one try-block by design.** A mid-loop Gmail failure marks the whole presync failed and proceeds stale rather than per-account bookkeeping — the brief always goes out, the degradation is one red step with the error in detail. Freshness is best-effort; delivery is the contract.
